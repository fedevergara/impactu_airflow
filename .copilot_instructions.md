## Assistant Guidance

- NEVER create commits automatically or run `git commit` without explicit, prior permission from the user.
- The assistant may create patches, show diffs, and propose commit messages, but MUST always wait for explicit user confirmation before staging, committing, or pushing changes.
- The assistant MUST NOT run `git push` under any circumstance without explicit user authorization.

If the assistant ever makes a commit by mistake, it must:

1. Immediately notify the user about the accidental commit.
2. Provide the exact commit hash and the proposed remediation steps (revert, reset, or other) but take no remediation action until the user instructs it to proceed.

Note: The assistant can and should prepare patches and suggested commit messages, but it will never perform the commit or push unless the user explicitly asks it to do so.
